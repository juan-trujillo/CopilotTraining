# GitHub Copilot in 15 Minutes

> **The Question This Talk Answers:**
> *"What is GitHub Copilot, where can my team use it, and how do we get started?"*

**Duration:** 15 minutes | **Target Audience:** Business-technical stakeholders

---

## 📊 Content Fitness

| Criterion | Assessment | Notes |
|-----------|-----------|-------|
| **Relevant** | 🟢 High | Essential first touchpoint for teams evaluating or rolling out Copilot |
| **Compelling** | 🟢 High | Focuses on surfaces (VS Code, CLI, GitHub.com) — concrete, not abstract |
| **Actionable** | 🟢 High | Ends with a clear today/this-week/this-month adoption roadmap |

---

## 📽️ Slide Sequence

1. **Title Slide** — AI-Powered Development in 15 Minutes
2. **What Is GitHub Copilot?** — How it works + business impact
3. **Three Surfaces, One AI** — VS Code · GitHub CLI · GitHub.com overview
4. **Surface 1: VS Code** — Completions, Chat, Agent Mode
5. **Surface 2: GitHub CLI** — Terminal AI, interactive & scripted
6. **Surface 3: GitHub.com** — Code Review, Cloud Agent, PR Chat
7. **How It All Fits Together** — Workflow integration + key principle
8. **Customization** — Instructions, agents, MCP, before/after
9. **Getting Started** — Today / This Week / This Month roadmap
10. **Thank You** — Same AI. Three surfaces. Your team's context.

---

## Key Messages

- **Same AI, three entry points** — Copilot meets teams where they already work
- **More context = better results** — the unifying principle across all surfaces
- **5 minutes to customize** — one markdown file transforms generic → team-specific
- **Progressive adoption** — start with VS Code today, expand to CLI and GitHub.com
